# Gate.io 美股数字货币 MA10 趋势监控系统 v2.0

## 一、项目简介

本项目是一个基于 **Gate.io U本位永续合约 API** 的 MA10（10周期移动平均线）趋势监控系统。系统支持 **Web 可视化面板** 和 **命令行脚本** 两种使用方式，可实时监控 100 个加密货币和美股代币的日K、4小时、60分钟、15分钟四个时间周期的 MA10 趋势方向（连续上涨 / 连续下跌 / 震荡）。

**核心特点：**
- 针对中国大陆网络环境优化（Gate.io 可直接访问，无需代理）
- 成交额前 100 标的自动筛选
- **模块化架构**：app.py / monitor.py / alerts.py / state.py / templates/index.html 五模块
- **技术指标**：MACD / RSI / 布林带 服务端+客户端双重计算
- **K线图表**：lightweight-charts 交互式图表，支持指标叠加
- **SSE 实时推送** + 60s 轮询降级
- **SQLite K线缓存**，API 调用量减半
- 亮色/暗色主题响应式 Web 面板，手机/电脑均可使用
- 价格提醒、键盘快捷键、骨架屏加载
- 零成本运行（Gate.io API 免费）

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 数据源 | Gate.io U本位合约 API | `https://api.gateio.ws/api/v4` |
| 后端 | Flask (Python) | 轻量 Web 框架，提供 REST API + SSE |
| 前端 | HTML5 + CSS3 + Vanilla JS | 暗色/亮色双主题、响应式网格、lightweight-charts K线图 |
| 图表 | lightweight-charts 4.1.3 | TradingView 出品，CDN 加载 |
| 数据存储 | JSON 文件 + SQLite + 内存缓存 | 原子写入防损坏，K线本地缓存 |
| 网络库 | requests (连接池复用) | 线程级 Session，并发5线程 |

## 四、快速开始

### 4.1 安装依赖

```powershell
pip install -r requirements.txt
```

或手动安装：

```powershell
pip install flask requests
```

### 4.2 启动 Web 面板（推荐）

```powershell
python app.py
```

启动后控制台会输出访问地址：

```
访问地址:
  本机: http://127.0.0.1:5000
  局域网: http://192.168.x.x:5000
```

**电脑浏览器访问：** `http://127.0.0.1:5000`

**手机访问（同一 WiFi 下）：** 打开手机浏览器，输入 `http://192.168.x.x:5000`（以启动时打印的实际 IP 为准）

### 4.3 添加到手机主屏幕（像 App 一样使用）

- **iPhone Safari**：打开网页 → 底部分享按钮 → "添加到主屏幕"
- **安卓 Chrome**：打开网页 → 右上角菜单 → "添加到主屏幕"

### 4.4 命令行版（备用）

```powershell
python gateio_futures_monitor.py
```

直接在终端输出所有标的的 MA10 趋势分析，适合查看原始数据或排查问题。

## 五、Web 面板功能说明

### 5.1 页面布局

| 区域 | 内容 |
|------|------|
| **顶部栏** | 标题 / 最后更新时间 / 手动刷新按钮 / 自动刷新倒计时 |
| **统计栏** | 日K上涨/下跌 / 4小时上涨/下跌 / 60分钟上涨/下跌 / 15分钟上涨/下跌 |
| **卡片网格** | 每个标的一张卡片，桌面多列、手机单列 |

### 5.2 卡片内容

每张卡片包含：
- **左上**：标的名称（如 BTCUSDT）+ 持仓方向标记（多/空/无，点击切换）
- **右上**：最新价 + 24h 涨跌幅 + 24h成交额（绿色涨 / 红色跌）
- **下方四格**：日K / 4小时 / 60分钟 / 15分钟

### 5.3 颜色编码

| 趋势状态 | 背景色 | 文字色 | 说明 |
|----------|--------|--------|------|
| 连续上涨 | 深绿色 | 绿色 | MA10 连续 >=3 周期上涨 |
| 连续下跌 | 深红色 | 红色 | MA10 连续 >=3 周期下跌 |
| 短期上涨 | 浅绿色 | 绿色 | MA10 1-2 周期上涨 |
| 短期下跌 | 浅红色 | 红色 | MA10 1-2 周期下跌 |
| 震荡 | 灰色 | 灰色 | MA10 无明显方向 |
| 数据不足 | 灰色 | 灰色 | K 线数据不足 20 根 |

### 5.4 刷新机制

- **自动刷新**：每 5 分钟（300秒）后台自动从 Gate.io 获取最新数据
- **手动刷新**：点击右上角"刷新数据"按钮，约 45 秒完成（32个标的 * 4次 API 请求）
- **缓存读取**：刷新期间前端仍显示旧数据，不会空白

## 七、周期背离信号（v1.3 新增）

### 7.1 筛选规则

系统在每个刷新周期自动检测**多时间框架背离**信号：

| 层级 | 大周期 | 小周期 | 信号条件 |
|------|--------|--------|----------|
| L1 | 日K | 4小时 | 日K连续 ≥5 周期同向，4小时反向 |
| L2 | 4小时 | 60分钟 | 4小时连续 ≥5 周期同向，60分钟反向 |
| L3 | 60分钟 | 15分钟 | 60分钟连续 ≥5 周期同向，15分钟反向 |

### 7.2 信号类型

- **买入信号**：大周期连续下跌 ≥5 周期，小周期上涨（潜在抄底机会）
- **卖出信号**：大周期连续上涨 ≥5 周期，小周期下跌（潜在见顶机会）

### 7.3 使用方式

1. **Web 面板**：点击顶部「周期背离」标签页查看所有当前符合条件的标的
2. **命令行**：运行 `python gateio_futures_monitor.py`，背离信号会在监控完成后输出

### 7.4 注意事项

- 背离信号仅表示**大周期趋势与小周期出现分歧**，不代表立即反转
- 建议结合持仓标记功能，将出现买入信号的标的重点关注
- 新合约数据不足时不会生成信号

## 九、关键注意事项

### 9.1 数据来源与准确性
- **Gate.io 合约价格 ≠ 真实股价**：美股代币合约（如 AMD_USDT）存在 **2-4% 的系统性溢价**
- **趋势方向可靠**：虽然绝对价格有溢价，但涨跌趋势与真实美股高度一致
- **新合约数据不足**：BILL、SOXL、DRAM、CBRS、QCOM 等日K历史数据较少，需等数据积累

### 9.2 网络环境
- **无需代理**：Gate.io API 在中国大陆可直接访问
- **Binance/OKX/CoinGecko/Yahoo Finance 不可用**：已全面测试确认超时或封禁
- **局域网访问限制**：Web 面板默认绑定 `0.0.0.0`，同一 WiFi 下可访问；跨网络需公网 IP 或云服务器

### 9.3 API 限流
- Gate.io API 有频率限制，脚本已内置 **0.25秒请求间隔**
- 单次全量刷新约 128 次 API 请求，耗时约 45 秒
- 避免短时间内多次点击"刷新数据"

## 十一、常见问题

**Q: 为什么有些标的一直显示"数据不足"？**
A: BILL_USDT、SOXL_USDT、DRAM_USDT、CBRS_USDT、QCOM_USDT 是 Gate.io 近期新上线的合约，日K历史数据不足 20 根。等 1-2 周数据积累后自动正常。

**Q: 为什么 AMD_USDT 价格和真实 AMD 股价不一样？**
A: Gate.io 上的美股代币是**股权代币永续合约**，价格由 Gate.io 市场供需决定，并非直接锚定真实股票交易所。实测存在 2-4% 溢价，但趋势方向一致。

**Q: 可以增加 NVDA、TSLA、AMZN 吗？**
A: Gate.io 目前没有这些合约。如需监控，需使用能访问美股数据源的方案（Finnhub/Alpha Vantage），但这些 API 在中国大陆可能也有访问限制。

**Q: 电脑关机后手机还能访问吗？**
A: 不能。当前方案是电脑本地运行 Flask 服务。如需 24h 访问，需部署到云服务器（约 20-50 元/月的香港/新加坡 VPS）。

**Q: 如何修改自动刷新间隔？**
A: 打开 `app.py`，找到 `AUTO_REFRESH_INTERVAL = 300`，将 300 改为需要的秒数（如 60 表示每分钟刷新）。
